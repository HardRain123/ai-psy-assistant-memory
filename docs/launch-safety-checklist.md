# 心理安全与合规上线清单

## 上线前必须完成

- 使用 PostgreSQL，启用 HTTPS，并轮换 Git 历史中曾出现过的后端共享令牌。
- FastAPI 与 Next 配置相同的 `BACKEND_SHARED_TOKEN`；Dify 只通过环境变量引用该值。
- 导入 `docs/psy-dsl-v0.17.yml`，确认风险枚举仍为 `low / medium / high`，没有 `urgent`。
- 配置 `WECHAT_WORK_WEBHOOK_URL`，用脱敏测试工单验证企业微信告警、失败重试和后台未通知状态。
- 配置 `BACKUP_DELETION_WEBHOOK_URL`。回调只接收删除申请 ID、用户哈希、删除证书编号和最晚删除日期，不接收聊天原文。
- 安排经过培训的安全运营人员，明确工作日 09:00–18:00（中国时间）的轮值和交接。
- 在 `/admin/safety` 验证确认、评估、联系、升级、解决、误报、复查时间和完整会话访问审计。
- 验证注册页六项分开授权；确认未满 18 岁不能注册。
- 验证手机号和紧急联系人功能保持关闭，数据库和页面不收集号码。

## 告警与处置演练

- `medium`：只进入复核队列，不发送企业微信告警，工作时段 30 分钟内查看。
- `high`：企业微信只包含事件 ID、风险等级、立即处置标志、时间和后台链接。
- `high + immediate_action_required=true`：用户端立即显示 110、120、远离危险物品、不要独处和联系可信任人员。
- 工作时段确认目标 5 分钟，首次人工响应目标 15 分钟。
- 非值守时段页面必须写明“无人实时查看”，工单排入下一工作日，不得声称有人已经看到。

## 删除与备份演练

- 用户申请删除后，账号立即冻结，所有登录会话立即撤销。
- 使用删除状态页确认 7 天计划删除时间，并验证撤回后可重新登录。
- 将测试申请的 `scheduled_for` 调整为过去时间，运行后台任务并确认主库数据已删除。
- 确认生成删除证书和删除清单，备份删除回调成功后 `backup_status=completed`。
- 模拟备份删除回调失败，确认任务会重试且邀请码自动暂停。

## 上线闸门

- `BETA_USER_LIMIT=50`，首批成人用户不得超过 50 人。
- 以下任一情况发生时停止发放邀请码：
  - 工作时段高风险工单超过首次响应截止时间。
  - 企业微信告警达到最大重试次数仍失败。
  - 主库删除任务失败。
  - 备份删除回调失败。
- 问题排除并完成复盘后，由管理员在 `/admin/safety` 填写原因并恢复邀请码。

## 备份删除回调契约

后端发送：

```json
{
  "request_id": "delete_xxx",
  "user_id_hash": "sha256-like-hmac",
  "certificate_id": "deletion_certificate_xxx",
  "delete_by": "ISO-8601",
  "action": "delete_user_from_all_backup_sets"
}
```

回调返回任意 `2xx` 表示基础设施已接受并完成删除清单；非 `2xx` 会记录失败、重试并触发上线闸门。
