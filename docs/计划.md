# 心理咨询 Agent 产品后续计划文档

## 1. 项目目标

本项目目标是先做出一个稳定、可持续陪伴用户的 AI 心理咨询助手，在咨询流程、风险处理、长期计划和用户数据管理可靠的基础上，再逐步商业化。

第一阶段不急着做复杂平台，而是先把核心能力做稳：

- 支持多次、长期、连续的心理咨询
- 根据每次咨询结果更新用户计划表
- 在需要时生成转介摘要，帮助用户转向真人咨询师或医疗资源
- 保障用户隐私、数据可控和风险安全

## 2. 产品阶段路线图

| 阶段 | 名称 | 核心目标 | 优先级 |
|---|---|---|---|
| P0 | 咨询流程稳定 | 把 AI 咨询主流程调通 | 最高 |
| P1 | 用户数据管理 | 支持查看、删除、导出记录 | 高 |
| P2 | 转介摘要 | 生成给真人咨询师看的交接材料 | 高 |
| P3 | 充值体系 | 控制模型成本，验证付费 | 中 |
| P4 | 咨询师目录 | 展示合规咨询师联系方式 | 中低 |
| P5 | 运营后台 | 投诉、风险、审核、数据看板 | 后续 |

## 3. P0：心理咨询流程稳定

### 目标

让用户能完成一次完整的 50 分钟咨询，并能多次回来继续之前的主题。

### 需要完成

- 新用户第一次进入，只说明一次 50 分钟咨询框架
- 老用户回来时，不重复完整开场
- 普通咨询能自然完成：开场、探索、整理、小行动、收尾
- 用户问“什么时候能改”时，必须给出小行动
- 用户问“还有多久结束”时，如果还剩超过 2 分钟，应继续聊
- 只有最后 1-2 分钟或用户主动结束时，才正式收束
- 每次咨询结束后更新 `care_plan`
- 高风险自伤场景必须进入安全干预流程

### 验收标准

- 连续 3 次咨询，总计 15 轮以上不中断
- 不重复机械开场
- 不在还剩 10 分钟时强行结束
- 能记住用户主要困扰、目标、阻碍和计划
- `care_plan` 每次结束后能滚动更新
- 不诊断、不开药、不承诺疗效

## 4. P1：用户数据管理

### 目标

让用户知道系统保存了什么，并能控制自己的数据。

### 需要完成

- 隐私说明
- 用户授权保存咨询记录
- 查看历史摘要
- 删除咨询记录
- 删除长期记忆
- 删除计划表
- 导出个人数据

### 关键注意

心理健康数据属于高度敏感信息，不能默认无限保存，也不能在用户不知情的情况下用于其他用途。
